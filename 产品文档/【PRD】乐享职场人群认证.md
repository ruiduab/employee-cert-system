# 【PRD】乐享职场人群认证

| 版本号 | 文档内容 | 时间 | 创建/更新人 |
|-------|---------|------|-----------|
| V1.0 | 支持乐享内完成职场人群身份认证，打通意图识别→认证→权益触达闭环 | 2026-04-17 | 产品团队 |

---

## 1. 需求背景

当前，乐享在企业会员场景已具备【企业会员认证】【企业钻石会员升级】等能力，覆盖了企业维度的认证需求。然而，针对**职场人群**（即在职企业员工的个人维度），乐享尚未提供身份认证能力。

职场人群认证旨在验证用户的**真实在职身份**（公司、职位、行业），从而提供差异化的产品优惠和定制化推荐：
- 基础职工认证用户可享受统一基础优惠
- 特定行业/职称用户（如法律、工业设计、开发等）可获得专项机型推荐和更深度优惠

当前痛点：
- 用户在乐享中咨询产品时，无法快速完成职工身份认证，需跳转外部页面
- 无法基于用户的职位/行业提供精准的产品推荐和差异化定价
- 职场人群的认证状态与乐享对话场景割裂，无法形成闭环体验

## 2. 需求目标

- **提升用户体验**：在乐享对话流程中无缝嵌入职场人群认证环节，减少用户跳转和操作成本
- **提高转化效率**：当用户表露产品咨询意图时，快速识别其职场身份，引导完成认证，第一时间展示专属优惠，刺激购买决策
- **拉动认证增长**：通过AI精准意图识别和便捷的认证流程，吸引更多职场用户完成身份认证
- **精准推荐**：基于用户的行业和职位信息，提供定制化的产品推荐

## 3. 需求描述

此功能涉及以下核心用户场景：
- **场景A**：乐享对话内触发职场人群认证
- **场景B**：PC商品详情页触发认证
- **场景C**：认证状态与进度查询
- **场景D**：认证后权益查询与产品推荐

## 4. 工具定义

### 4.1 前置工具：职场认证状态查询工具

- **工具名称**：【职场人群认证状态查询】
- **工具用途**：实时判断当前用户的职场身份认证状态，为后续场景分流提供依据
- **调用时机**：当用户query被识别为与职场认证相关的意图时，优先调用此工具获取用户状态
- **返回状态**：

| 状态 | 说明 |
|------|------|
| 未登录 | 用户未登录（正常跳转登录，保持乐享现有逻辑） |
| 未认证 | 已登录，但未完成职场人群认证 |
| 基础认证 | 已通过基础职工身份认证，享受基础优惠 |
| 专项认证 | 已通过特定行业/职称认证（返回具体认证类型：法律/工业设计/创意设计/开发编程） |
| 审核中 | 认证申请已提交，正在审核队列中 |
| 认证过期 | 认证已过有效期（2年），需重新认证 |

- **返回字段**：

```json
{
  "status": "专项认证",
  "cert_types": ["basic", "legal"],
  "company": "XX科技有限公司",
  "job_title": "法律顾问",
  "industry": "互联网/信息技术",
  "cert_time": "2025-04-10",
  "expire_time": "2027-04-10",
  "privileges": [
    {"type": "basic", "discount": 0.95, "scope": "全系产品"},
    {"type": "legal", "discount": 0.85, "scope": "Carbon系列"}
  ]
}
```

### 4.2 核心工具：职场人群认证工具

- **工具名称**：【职场人群认证】
- **工具用途**：可被多场景调用的标准化认证模块，在用户未认证时引导完成职场身份认证
- **工具特性**：
  - 工具导向设计：不绑定特定意图，任何场景需要验证用户职场身份时均可调用
  - 唤起认证表单页面（H5/PC），完成从信息采集到材料提交的完整流程
- **调用方式**：在回复下方展示【立即认证】按钮，用户点击后唤起认证表单

### 4.3 辅助工具：认证审核进度查询工具

- **工具名称**：【职场认证审核进度查询】
- **工具用途**：查询用户提交的职场认证申请的审核状态
- **返回状态**：

| 状态 | 说明 |
|------|------|
| 未提交 | 系统中无该用户的待审核申请 |
| 待审核 | 申请已提交，在人工审核队列中 |
| 审核通过 | 认证已通过（此时【认证状态查询工具】同步更新） |
| 审核未通过 | 未通过审核，返回驳回原因 |

- **返回字段**：

```json
{
  "status": "审核未通过",
  "reject_reason": "上传的在职证明截图不清晰",
  "submitted_at": "2026-04-15T10:30:00Z",
  "reviewed_at": "2026-04-16T14:00:00Z"
}
```

## 5. 场景A：乐享对话内触发认证

### 5.1 触发条件

当用户在乐享对话框中输入的query，通过意图理解被判定为以下意图之一时触发：

**意图一：职场认证直接意图**
- 精准意图示例：
  - "我要认证职工身份"
  - "如何进行职场认证"
  - "怎么认证在职身份"
  - "我想做职工认证"
  - "职场人群认证怎么弄"
- 模糊意图示例：
  - "在职员工有优惠吗"
  - "公司员工能打折吗"
  - "职工专属价怎么获得"
  - "我是XX公司的员工"

**意图二：产品咨询+职场身份关联意图**
- 意图示例：
  - "我是律师，想买个办公笔记本"
  - "我做工业设计的，推荐一款电脑"
  - "程序员用什么笔记本好"
  - "设计师适合用哪款"
  - "我在XX公司做开发，有推荐吗"

> **注意**：与【企业会员认证】的意图区分：
> - "企业采购""批量购买""企业优惠" → 触发企业会员认证
> - "职工认证""员工优惠""个人职场身份" → 触发职场人群认证
> - 当意图模糊时（如"在职员工买电脑有优惠吗"），优先触发职场人群认证

### 5.2 交互流程

#### 步骤一：意图识别与身份判断

乐享识别到相关意图后，调用【职场人群认证状态查询】工具确认当前用户状态。

#### 步骤二：场景分类

**情况一：用户未认证**

- 核心逻辑：引导用户进行职场身份认证，以解锁专属权益
- 示例话术：
  - （直接意图）"您好！联想为在职员工提供专属的职场认证权益。完成认证后，您可以享受产品优惠折扣、专属推荐等权益。点击下方按钮即可快速完成认证。"
  - （产品咨询意图）"您好！针对您的需求，联想有多款适合的产品。检测到您尚未完成职场身份认证，完成认证后可享受专属优惠价格和定制推荐。"
- 交互动作：在回复下方展示【立即认证职场身份】按钮

**情况二：用户已认证（基础认证）**

- 核心逻辑：正常回答 + 基础权益展示 + 专项认证升级引导
- 示例话术：
  - （直接意图）"您好！您已完成基础职工身份认证，当前可享受全系产品95折优惠。如果您的职位属于特定行业（法律、工业设计、创意设计、开发编程），还可以申请专项认证，享受更深度的优惠和专属机型推荐。"
  - （产品咨询意图）"您好，已认证职工！针对您的需求，为您推荐以下产品（已享受职工95折优惠价）……如果您的职业属于法律、设计、开发等领域，可申请专项认证获得更大优惠。"
- 交互动作：在回复下方展示【申请专项认证】按钮（引导至专项认证补充流程）

**情况三：用户已认证（专项认证）**

- 核心逻辑：正常回答 + 强化专项权益感知 + 精准产品推荐
- 示例话术：
  - （直接意图）"您好！您已通过[法律专业]认证，当前可享受：全系产品95折基础优惠 + Carbon系列85折专项优惠。您的认证有效期至2027年4月10日。"
  - （产品咨询意图）"尊敬的已认证[法律专业]用户，您好！根据您的职业特点，为您推荐ThinkPad X1 Carbon，专项优惠价¥11,049（原价¥12,999），轻薄便携，非常适合法律工作者。"
- 交互动作：调用【职场认证权益查询】工具，展示个性化推荐产品卡片

**情况四：用户认证审核中**

- 核心逻辑：告知审核进度，安抚等待
- 示例话术："您好！您的职场身份认证申请正在审核中，我们会在3个工作日内完成审核。审核通过后您即可享受专属权益，请耐心等待。"

**情况五：用户认证已过期**

- 核心逻辑：引导重新认证
- 示例话术："您好！您的职场身份认证已于[2027年4月10日]到期。为继续享受专属权益，请点击下方按钮重新提交认证。"
- 交互动作：在回复下方展示【重新认证】按钮

### 5.3 认证表单唤起与提交

用户点击【立即认证职场身份】按钮后：

1. **唤起认证表单**：弹出认证H5页面（移动端）或侧边面板（PC端）
2. **表单流程**（共4步）：

| 步骤 | 内容 | 说明 |
|------|------|------|
| 第一步 | 企业信息 | 搜索/选择企业名称，自动带出统一社会信用代码 |
| 第二步 | 职位信息 | 填写/选择所在职位，选择所属行业 |
| 第三步 | 认证材料 | 四选一：企业邮箱认证（推荐）/ 个税APP截图 / 社保截图 / 工卡名片等 |
| 第四步 | 提交认证 | 确认信息，提交审核 |

3. **提交后自动发送query**：表单提交成功后，前端自动向乐享发送query"提交职场身份认证"
4. **乐享回复**：
   - 示例话术："您的职场身份认证申请已成功提交！我们会在3个工作日内完成审核，届时将通知您审核结果。审核通过后，您即可享受职工专属权益。"

## 6. 场景B：PC商品详情页触发认证

### 6.1 触发条件

- **触发条件A**：用户在PC商品详情页，唤起乐享侧边栏后，默认首条answer包含职场认证信息
- **触发条件B**：商品详情页设有"职工专属价"标签，点击后引导认证

### 6.2 交互流程

1. **用户操作**：用户在商品详情页看到"职工专属价"提示，或点击"查看职工优惠"入口
2. **身份判断**：
   - 情况一：用户已认证 → 直接展示职工专属价格
   - 情况二：用户未认证 → 显示引导认证的文案和按钮
     - 示例文案："完成职场身份认证，立享专属优惠价"
     - 展示【立即认证】按钮
3. **唤起乐享**：
   - 用户点击【立即认证】按钮
   - 页面右侧边栏自动滑出乐享AI
   - 前端自动向乐享发送预设query："申请职场身份认证"
4. **进入认证流程**：与场景A逻辑一致

## 7. 场景C：认证状态与进度查询

### 7.1 触发条件

当用户的query被判定为具有"查询认证状态""认证进度""审核结果"等意图时触发。

**意图关键词示例**：
- "我的职工认证审核了吗"
- "认证进度查询"
- "认证通过了没有"
- "职场认证状态"
- "认证什么时候能好"
- "我的认证被驳回了怎么办"

### 7.2 交互流程

乐享识别到意图后，调用【职场认证审核进度查询】工具获取状态。

**情况一：未提交过申请**

- 判断用户是否已登录及认证前置状态
- 示例话术："您好，系统未查询到您的职场认证申请记录。如需认证，可点击下方按钮开始认证。"
- 交互动作：展示【立即认证职场身份】按钮

**情况二：待审核**

- 示例话术："您的职场身份认证申请已收到，目前正在审核中。我们会在3个工作日内完成审核，请您耐心等待。"

**情况三：审核通过**

- 示例话术："恭喜您！您的职场身份认证已通过。您当前的认证类型为[基础职工认证+法律专业认证]，可享受全系产品95折和Carbon系列85折优惠。"
- 交互动作：调用【职场认证权益查询】工具，展示权益详情

**情况四：审核未通过（有原因）**

- 示例话术："很抱歉，您的职场身份认证未通过。驳回原因：[上传的在职证明截图不清晰]。您可以准备好材料后，点击下方按钮重新提交认证。"
- 交互动作：展示【重新认证】按钮

**情况五：审核未通过（无原因）**

- 示例话术："很抱歉，您的职场身份认证未通过。您可以检查提交的材料是否清晰完整，然后点击下方按钮重新提交。"
- 交互动作：展示【重新认证】按钮

- 核心逻辑：展示完整权益信息和推荐产品

## 9. 知识库配置

乐享需配置以下知识内容，用于回答用户关于职场认证的常见问题：

### 9.1 认证流程相关知识

| 用户问题 | 回答要点 |
|---------|---------|
| "职场认证需要什么材料？" | 基础认证需提供企业信息+职位信息+认证材料（企业邮箱/个税截图/社保截图/工卡名片任选一种） |
| "认证需要多长时间？" | 提交后3个工作日内完成审核 |
| "认证有效期多久？" | 认证有效期为2年，到期前30天提醒续期 |
| "认证被驳回了怎么办？" | 可根据驳回原因补充材料重新提交，或在7天内提交申诉 |
| "什么是专项认证？" | 针对法律、工业设计、创意设计、开发编程等特定职业的深度认证，可获得专属机型推荐和更大优惠 |
| "基础认证和专项认证有什么区别？" | 基础认证：全系产品95折；专项认证：在基础折扣上叠加特定产品线更深优惠（最低82折） |

### 9.2 权益相关知识

## 11. 与现有功能的关系

| 功能 | 维度 | 关系 |
|------|------|------|
| 企业会员认证 | 企业维度 | 互补关系。企业会员认证面向企业采购场景；职场人群认证面向个人职业身份场景。一个用户可同时拥有企业会员身份和职场认证身份 |
| 企业钻石会员升级 | 企业维度 | 无冲突。钻石会员升级是企业会员体系内的等级提升；职场认证是独立的个人身份认证体系 |
| 职场人群认证 | 个人维度 | 本功能。基于个人的公司、职位、行业进行身份认证 |

**意图区分规则**：
- 含"企业采购""批量购买""企业会员"关键词 → 企业会员认证
- 含"升级钻石""提升等级"关键词 → 钻石会员升级
- 含"职工认证""员工优惠""我是[职业]""个人认证"关键词 → 职场人群认证
- 模糊场景（如用户仅说"认证"）→ 先调用两个状态查询工具判断，根据用户已有状态引导

## 12. 验收标准

| 模块 | 验收标准 |
|------|---------|
| 意图识别 | 对于明确的职场认证意图（如"职工认证""我是律师想买电脑"），乐享必须能够准确识别并触发对应工具调用 |
| 用户状态隔离 | 未认证用户必须看到认证引导和按钮；已认证用户必须直接获得权益展示和专属推荐 |
| 工具调用准确 | 每个场景的工具调用链路必须正确：先查状态 → 再决定后续工具调用 |
| 认证表单唤起 | 点击认证按钮后必须成功唤起认证表单页面，表单流程完整可用 |
| 自动query发送 | 认证表单提交成功后，必须自动向乐享发送"提交职场身份认证"query并触发确认回复 |
| 进度查询 | 四种审核状态（未提交/待审核/通过/未通过）均必须返回正确的话术和交互动作 |
| 权益展示 | 已认证用户查询权益时，必须展示完整的优惠信息、推荐产品和认证有效期 |
| 与企业会员意图隔离 | 职场认证意图与企业会员认证意图必须正确区分，不发生误触发 |
| PC商品页联动 | 商品详情页的认证引导按钮必须能成功唤起乐享并自动发送认证query |
| 错误处理 | 工具调用失败时必须给出友好提示（如"网络繁忙，请稍后重试"），不能出现无响应 |
