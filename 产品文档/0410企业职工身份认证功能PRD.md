## 2. 产品概述

### 2.1 背景与意义
针对企业在职职工用户群体，提供身份认证功能，验证用户真实职工身份。通过分层认证机制，为不同职务、行业的职工提供差异化的产品优惠和定制化的产品推荐，提升转化率和客户满意度。

### 2.2 核心价值
- **用户侧**：快速获得身份认证，享受职工专属优惠
- **商业侧**：精准识别目标用户，提供差异化定价策略，增加企业采购转化
- **数据侧**：积累职工身份数据，支撑精准营销和产品推荐

## 3. 产品功能

### 3.1 认证体系

#### 第一层：基础职员身份认证
- **认证范围**：任何在职企业员工
- **认证不要求**：特定行业、职称、职位
- **认证权益**：统一的基础优惠（如95折）
- **验证方式**：
  - 实名认证 + 身份证
  - 在职公司及职位认证
  - 企业邮箱认证（可选）

#### 第二层：行业/职称专项认证
- **认证范围**：特定行业或职称的员工
- **认证权益**：专项机型优惠与定制推荐
  
| 认证类型 | 适用人群 | 专属机型 | 优惠幅度 |
|---------|---------|---------|---------|
| 法律专业认证 | 律师、法务、法律咨询 | Carbon系列 | 根据商务制定 |
| 工业设计认证 | 工业设计师、产品设计师 | ThinkPad P系列（P900c等） | 根据商务制定 |
| 创意设计认证 | 媒体设计、视觉设计、UI设计 | ThinkPad 16p | 根据商务制定 |
| 开发编程认证 | 程序员、开发工程师、算法工程师 | ThinkPad T14p | 根据商务制定 |

### 3.2 用户认证流程

```
开始
  ↓
用户登录 → 进入个人中心 → 点击"申请职工认证"
  ↓
第一步：选择认证类型
  ├─ 基础职员认证
  └─ 行业/职称认证（可多选）
  ↓
第二步：实名认证（基础认证必需）
  ├─ 填写真实姓名
  ├─ 填写身份证号
  ├─ 提交人脸识别（可选，提高通过率）
  └─ 身份验证接口调用
  ↓
第三步：在职信息认证（基础认证必需）
  ├─ 选择公司 / 搜索公司
  ├─ 填写/选择职位信息
  ├─ 填写/选择所属行业（若申报专项认证）
  ├─ 企业信息验证接口调用
  └─ 职位行业信息存储
  ↓
第四步：认证材料提交（基础认证）
  ├─ 选择任意一种方式：
  │  ├─ 企业邮箱认证（推荐）
  │  ├─ 劳动合同/在职证明
  │  ├─ 社保缴纳截图
  │  └─ 其他在职证明材料
  ├─ 企业邮箱验证接口调用（若选择此方式）
  └─ 材料上传存储
  ↓
第五步：专项认证补充（仅限申报专项认证用户）
  ├─ 行业/职位验证
  ├─ 职业资格认证（可选）：
  │  ├─ 二维码名片认证
  │  ├─ 个人所得税APP认证
  │  └─ 职业资格证书/奖项
  └─ 材料上传验证接口调用
  ↓
审核结果
  ├─ 通过 → 认证成功 → 签发认证证书 → 享受权益
  └─ 驳回 → 显示驳回原因 → 可重新提交
  ↓
结束
```

## 5. 业务规则与验证

### 5.1 基础职员认证规则
- **必要条件**：
  - ✓ 完成实名认证（身份证验证通过）
  - ✓ 完成在职认证（公司+职位）
  - ✓ 选择一种认证材料方式并通过验证
  
- **验证逻辑**：
  1. 身份证有效期检查
  2. 企业库匹配验证
  3. 邮箱域名验证（若选择邮箱认证）
  4. 材料真实性人工审核

- **有效期**：
  - 认证有效期：2年
  - 需每2年重新提交审核（提前30天提醒）

### 5.2 专项认证规则
- **额外条件**：
  - ✓ 完成基础认证
  - ✓ 职位/行业信息精确匹配专项认证类型
  - ✓ 提交相关职业资格证明或材料

- **验证逻辑**：
  - 职位关键词匹配（模糊匹配 + 人工审核）
  - 资格证书验证（社保/个税数据查询）
  - 材料真实性检查

- **有效期**：
  - 与基础认证保持一致（2年）

### 5.3 驳回与申诉
- **常见驳回原因**：
  - 身份信息不一致
  - 企业无法验证
  - 材料不清晰或无效
  - 职位信息与专项认证不符
  
- **申诉流程**：
  - 用户可在驳回后7天内申诉一次
  - 人工审核，3-5个工作日反馈
  - 可重新提交所有材料

#### 关键业务流程

**1. 认证流程**
```java
// 伪代码
public CertificationResult submitCertification(CertificationRequest req) {
    // 1. 实名认证
    RealNameResult = verifyRealName(req.idCard, req.idName);
    
    // 2. 企业认证
    CompanyResult = verifyCompany(req.company, req.jobTitle);
    
    // 3. 材料认证
    MaterialResult = verifyMaterial(req.material, req.materialType);
    
    // 4. 综合判断
    if (RealNameResult.pass && CompanyResult.pass && MaterialResult.pass) {
        createCertification(user_id, cert_type);
        return SUCCESS;
    } else {
        recordRejection(user_id, reason);
        return FAILED;
    }
}
```

**2. 权益查询流程**
```java
public UserPrivilege getUserPrivilege(Long userId) {
    // 1. 查询用户认证状态
    Certification cert = getCertification(userId);
    
    // 2. 根据认证类型获取权益
    Privilege priv = getPrivilege(cert.cert_type);
    
    // 3. 计算优惠后价格
    Price discountedPrice = calculatePrice(
        originalPrice, 
        priv.discount_rate
    );
    
    return new UserPrivilege(priv, discountedPrice);
}
```

## 9. 安全与风险控制

### 9.1 数据安全
- **身份证号加密**：使用AES-256加密存储，仅后端可解密
- **个人信息**：GDPR/DPA合规，用户可申请删除认证数据
- **材料存储**：使用阿里云/腾讯云OSS，自动清理超期数据（3个月）
- **传输安全**：所有敏感数据使用HTTPS传输

### 9.2 风险控制
- **频率限制**：
  - 同一用户24小时内最多提交3次认证申请
  - 邮箱验证码5分钟有效期，最多发送3次
  - IP黑名单管理

- **异常检测**：
  - 大量短时间内认证申请（风险评分提升）
  - 虚假企业信息识别（黑名单库）
  - 材料PS识别（AI检测）

- **人工审核**：
  - 所有基础认证通过人工最终审核
  - 高风险申请需要双重审批
  - 可疑材料进入人工审查队列

### 9.3 隐私保护
- 前端不传输完整身份证号（仅传输后6位或加密）
- 个人信息分级展示（列表隐藏，详情可见）
- 审计日志记录所有操作（6个月保留期）
- 定期进行安全审计与渗透测试

### B. 常见问题

**Q: 用户认证失败如何重试？**
A: 用户可在驳回后立即重新提交一次，或在7天后提交申诉，由人工审核后可重试。

**Q: 认证数据如何备份和迁移？**
A: 每日自动备份到云端，支持数据导出，满足GDPR要求。

**Q: 专项认证权益如何联动促销？**
A: 根据用户认证类型，在个性化推荐、邮件营销、APP Push中动态调整优惠信息。
