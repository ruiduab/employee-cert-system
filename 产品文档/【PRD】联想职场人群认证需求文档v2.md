# 一、产品概述

联想职工认证是一套面向企业员工的身份认证体系，通过4个步骤的标准化流程，完成员工身份验证并授予相应权益。

认证完成后，用户获得统一的**职场人群认证身份**，不区分行业或职称等级。

> 生产环境：联想官网 + 乐享智能体

---

# 二、认证基础条件（关键前置条件）

用户必须同时满足以下两个条件才能进行职工身份认证：

## 2.1 条件一：用户登录

- 用户必须完成登录：用户访问页面点击认证，判断用户是否登录，未登录用户跳转到对应的登录页面
- 登录后系统识别用户身份走不同流程：
  - **个人用户**：实名成功后 → 提示一键升级成为企业用户，发放企业权益礼包和职场认证礼包
  - **企业用户**：实名过程中带出企业信息不支持修改，仅填写岗位信息，认证成功后发放职场认证礼包

## 2.2 条件二：完成个人实名认证

- 用户必须完成个人实名认证（第2步）
- 这是进行任何职工身份认证的必要前置条件
- 系统通过 `hasRealname` 字段跟踪认证状态

**关键说明**：个人实名认证（第2步）与职工身份认证（第3、4步）分开：
- 个人实名认证：验证个人身份的真实性
- 职工身份认证：验证企业员工身份

# 三、认证流程（核心交互逻辑）

## 3.1 流程决策树

```
用户进入系统
  ↓
显示第1步：默认认证类型（企业职工认证）
  ↓
[系统判断] hasRealname 状态？
├─ hasRealname = false（未完成个人实名认证）
│  ├─ 弹窗提示："你还未完成个人实名认证，需要先完成实名认证才能进行职工认证"
│  ├─ 用户选择"继续进行实名认证" → 进入第2步（未完成状态）
│  ├─ 完成实名认证 → 系统标记 hasRealname = true
│  └─ → 自动进入第3步
│
└─ hasRealname = true（已完成个人实名认证）
   ├─ 无弹窗提示
   ├─ 直接进入第3步
   └─ 第2步显示"已完成"状态（可返回查看）
```

# 四、详细交互规则

## 4.1 第1步：选择认证类型

**页面显示内容**：
- 显示认证入口及当前设计的offering（offering待确认）
- 仅展示"企业职工认证"一种类型

**交互流程**：

### A. 用户尚未完成个人实名认证（hasRealname = false）

1. 用户在第1步看到认证类型（企业职工认证，默认选中）
2. 点击"下一步"按钮 → 系统触发前置检查
3. 系统检查 `hasRealname` 状态，发现为 false → 弹窗提示
4. 弹窗内容：
   - 标题："完成实名认证"
   - 内容："你还未完成个人实名认证，需要先完成实名认证才能进行职工认证。"
   - 按钮选项：
     - ✓ "继续进行实名认证" → 跳转至第2步
     - ✗ "返回" → 留在第1步

### B. 用户已完成个人实名认证（hasRealname = true）

1. 用户在第1步看到认证类型
2. 点击"下一步"按钮 → 系统检查通过
3. 直接进入第3步（无弹窗提示）
4. 第2步已完成状态被跳过，但用户可通过返回按钮查看

## 4.2 第2步：个人实名认证

**作用**：验证用户身份的真实性，确保身份信息真实有效

**隐私保护**：
- 身份信息将被加密存储
- 仅用于身份验证，不会泄露给第三方

### 状态一：未完成状态（首次进入）

**显示内容**：
- 页面标题："第2步：实名认证"
- 说明文字："选择认证方式验证你的身份信息"
- 认证方式：身份证认证
  - 所需信息：真实姓名（2-20个汉字）+ 身份证号码（18位）
  - 说明："请输入您的真实身份信息。身份证号码需为18位，请确保信息准确无误。"

**流程**：
1. 默认手机号：进入第2步时，手机号字段自动填充（模拟从注册账号获取）
2. 修改手机号：用户可直接编辑手机号，系统实时验证格式
3. 发送验证码：点击"发送验证码"按钮，验证成功弹窗提示
4. 完成认证：填写所有字段 + 勾选协议 → 点"下一步"进入第3步

**系统操作**：
```
当用户在第2步点击"下一步"时：
1. 系统标记 hasRealname = true
2. 自动跳过第2步，直接显示第3步
```

**错误提示**：
- 姓名为空：请输入姓名
- 姓名格式错误：姓名必须为2-20个汉字
- 身份证号为空：请输入身份证号
- 身份证号格式错误：身份证号必须为18位数字
- 手机号为空：请输入手机号
- 手机号格式错误：请输入有效的手机号
- 验证码为空：请输入验证码
- 验证码格式错误：验证码为6位数字
- 未勾选协议：请同意实名认证协议

### 状态二：已完成状态（返回时）

**显示内容**：
- ✅ 提示文案："实名认证已完成"
- 说明："你已完成个人实名认证，可以继续进行职工认证"

**关键机制**：
- 第2步一旦标记为已完成（hasRealname = true），无法再次修改
- 用户返回到第2步时，始终显示已完成状态

## 4.3 第3步：企业在职认证

**显示触发条件**：用户从第1步完成判断后进入

**所需信息**：

1. **企业名称**（必填）
   - 严格校验天眼查数据，支持中文搜索点击选择，不支持自定义输入修改
   - 如当前用户为企业购注册用户，自动带出注册时的企业名称，不支持修改
   - 填写2个中文字符即调用天眼查接口，提供企业全称供用户单击选择

2. **企业验证状态**：
   - 选择企业后，系统验证该企业是否有固定邮箱后缀
   - ✅ 企业信息已确认（支持邮箱认证）— 该企业有企业邮箱后缀（如：lenovo.com）
   - 验证状态影响第4步：有已知邮箱后缀则自动填入，无则用户手动输入

3. **职位信息**（必填）— 2-12字符，用户填写匹配出职业列表，点击选择，支持自定义职位

4. **所属行业**（必填）— 下拉选择（对接天眼查行业信息表）

> ⚠️ 行业列表和职位信息规范待提供

## 4.4 第4步：选择在职证明方式

说明：用户可从以下四种方式中任选其一，提交在职身份证明。

### 方式1：企业邮箱认证（推荐）

这是自动化率最高、体验最顺畅的路径，应当作为主推方式。

**所需材料**：用户输入企业邮箱地址【必填】（如 name@company.com）

**系统验证方案（全自动）**：

1. **域名校验**：后端维护一份"公共邮箱后缀黑名单"（如 @qq.com、@163.com、@gmail.com 等）。用户输入邮箱后，系统实时校验后缀，若在黑名单内，前端直接拦截并提示不支持。

2. **邮箱后缀处理**：
   - 分离输入框：用户名 + 邮箱后缀
   - 如第3步企业有已知邮箱后缀 → 自动填入，后缀字段只读
   - 如第3步企业无已知邮箱后缀 → 用户手动输入后缀

3. **OTP 验证**：向该邮箱发送6位动态验证码，用户回填正确即刻通过认证。

4. **认证结果**：邮箱验证码校验通过 → 认证成功，即时生效。

**适用场景**：绝大多数互联网、科技、金融及规上企业的员工。

### 方式2：劳动合同或在职证明

针对没有企业邮箱或所在企业不规范的用户群体。

**所需材料（二选一）**：
- 在职证明：带有姓名（与实名认证一致）、企业名称、明确有效期的证明文件照片
- 劳动合同：包含甲乙双方基本信息页、盖章签字页的照片

**系统验证方案（OCR自动审核）**：

1. **AI信息抽取**：调用视觉Agent对上传的图片进行预处理（防翻拍检测），结构化提取核心字段：姓名、公司名称、职位。

2. **交叉核验**：
   - 比对提取的【姓名】与用户【实名认证信息】是否一致
   - 比对提取的【公司名称】与第3步填写的企业名称是否一致

3. **印章检测（能力待确认）**：检测画面中是否存在红色印章，提取印章内公司全称，与文本提取的公司名进行二次交叉比对。

4. **认证结果**：
   - OCR通过（置信度 > 95%）→ **认证成功**，即时生效
   - OCR不通过 → **认证失败**，提示用户重新提交或联系客服

**所需**：文件上传（JPG/PNG/PDF，5MB以内）

### 方式3：个人所得税APP认证

个税APP的界面是国家统一标准，格式固定，适合纯自动化处理。

**所需材料**：用户在"个人所得税APP"中，"任职受雇信息"详情页的完整截图。

**系统验证方案（模板化OCR，极高自动化）**：

1. **特征点锚定**：系统预设个税APP界面的标准特征模板。
2. **关键字段捕获**：
   - 【姓名】（与实名比对）
   - 【扣缴义务人】（即公司名称，全称提取）
   - 【任职受雇从业类型】（需为"雇员"等正常状态）
   - 【状态】（需提取到"正常"字样）
3. **防伪检测**：检测截图是否存在拼接痕迹或字体不一的PS痕迹。
4. **认证结果**：
   - OCR通过 → **认证成功**，即时生效
   - OCR不通过 → **认证失败**，提示用户重新提交或联系客服

**所需**：文件上传（JPG/PNG/PDF，5MB以内）

### 方式4：其他材料认证（兜底方案）

**所需材料**：支持多图上传，例如：社保缴纳记录截图、带有公司Logo和姓名的工牌、名片等。

**系统验证方案（OCR自动审核）**：

1. **分类路由**：系统先判断图片类型（是社保截图、证件还是工牌）
2. **要素提取**：AI提取图片里的"人名"和"机构名"，校验人名与实名信息一致
3. **认证结果**：
   - OCR通过 → **认证成功**，即时生效
   - OCR不通过 → **认证失败**，提示用户重新提交或联系客服

**所需**：文件上传（JPG/PNG/PDF，5MB以内）

# 五、审核机制说明（重要）

## 5.1 审核流程概述

本系统**不设人工审核环节**，所有认证均为自动化处理：

| 认证方式 | 审核机制 | 结果 |
|---------|---------|------|
| 企业邮箱 | 邮箱验证码校验（全自动） | 验证码正确 → 即时通过 |
| 劳动合同/在职证明 | OCR识别+交叉核验（全自动） | 通过/不通过 |
| 个人所得税APP | 模板化OCR识别（全自动） | 通过/不通过 |
| 其他材料 | OCR识别+要素提取（全自动） | 通过/不通过 |

## 5.2 认证状态

| 状态 | 说明 |
|------|------|
| 未认证 | 用户未提交认证申请 |
| 邮箱待验证 | 用户选择邮箱认证，验证码已发送，等待用户回填验证（仅邮箱认证场景） |
| 认证成功 | 邮箱验证通过 或 OCR审核通过，认证即时生效 |
| 认证失败 | OCR审核未通过，用户可重新提交或联系客服 |
| 已失效 | 认证有效期（365天）到期，用户需重新认证 |

## 5.3 认证失败处理流程

```
用户提交材料 → OCR自动审核
  ├─ 通过 → 认证成功（即时生效）
  └─ 不通过 → 认证失败
       ├─ 用户可重新提交材料认证
       └─ 用户联系客服 → 客服人员进入后台审核列表 → 修改被驳回的状态为通过
```

**关键说明**：后台审核管理的作用是为客服/业务人员提供修改被驳回状态的入口，而非日常审核流程。

# 六、关键判断逻辑总结

## 6.1 系统状态跟踪

系统通过以下字段跟踪用户状态：
- `hasRealname`：个人实名认证状态（true/false）
- `currentStep`：当前步骤（1-5）
- `certStatus`：认证状态（未认证/邮箱待验证/认证成功/认证失败/已失效）

## 6.2 第1步的判断逻辑

```
用户点击"下一步"时：
判断：检查 hasRealname
├─ false → 显示弹窗，引导完成实名认证
└─ true → 直接进入第3步
```

## 6.3 第2步的判断逻辑

```
用户点击"下一步"时：
步骤1：标记 hasRealname = true
步骤2：直接进入第3步
```

## 6.4 返回按钮逻辑

```
用户点击"返回"时：
- 从第4步返回 → 进入第3步
- 从第3步返回 → 进入第2步（显示已完成状态）
- 从第2步返回 → 进入第1步
- 从第1步返回 → 无操作（已在第1步）
```

# 八、用户交互流程演示

## 场景一：首次使用（未完成实名认证）

```
1. 用户进入第1步
   - hasRealname = false
   - 默认认证类型：企业职工认证
2. 点击"下一步"
   - 系统检查发现 hasRealname = false
   - 显示弹窗提示
3. 点击"继续进行实名认证"
   - 进入第2步（未完成状态）
   - 显示个人实名认证方式：身份证认证，输入姓名和身份证号
4. 点击"下一步"
   - 系统标记 hasRealname = true
   - 自动进入第3步（企业在职认证）
5. 填写企业信息，继续认证流程
   - 第3步 → 第4步 → 提交
6. 提交认证
   - 邮箱认证：发送验证码 → 用户回填 → 即时通过
   - 其他方式：OCR自动审核 → 通过/不通过
```

## 场景二：已完成实名认证

```
1. 用户进入第1步
   - hasRealname = true（系统已记录）
2. 点击"下一步"
   - 系统检查发现 hasRealname = true
   - 无弹窗提示，直接进入第3步
3. 填写企业信息 → 第4步 → 提交 → 完成
```

# 九、乐享智能体集成

## 9.1 核心场景

| 场景 | 说明 |
|------|------|
| 场景A | 乐享对话内触发职场人群认证 |
| 场景B | 活动页、活动入口触发认证 |
| 场景C | 认证状态与进度查询 |
| 场景D | 认证后权益查询与产品推荐（待定） |

## 9.2 工具定义

### 9.2.1 前置工具：职场认证状态查询工具

- **工具名称**：【职场人群认证状态查询】
- **工具用途**：实时判断当前用户的职场身份认证状态，为后续场景分流提供依据
- **调用时机**：当用户query被识别为与职场认证相关的意图时，优先调用此工具获取用户状态
- **返回状态**：

| 状态 | 说明 |
|------|------|
| 未登录 | 用户未登录（正常跳转登录，保持乐享现有逻辑） |
| 未认证 | 已登录，但未完成职场人群认证 |
| 邮箱待验证 | 已提交邮箱认证，验证码已发送，等待用户回填验证 |
| 认证成功 | 已通过认证（邮箱验证通过 或 OCR通过） |
| 认证失败 | OCR审核未通过 |
| 已失效 | 认证有效期（365天）已过期 |

### 9.2.2 核心工具：职场人群认证工具

- **工具名称**：【职场人群认证】
- **工具用途**：可被多场景调用的标准化认证模块，在用户未认证时引导完成职场身份认证
- **工具特性**：
  - 工具导向设计：不绑定特定意图，任何场景需要验证用户职场身份时均可调用
  - 唤起认证表单弹窗（H5/PC），完成从信息采集到材料提交的完整流程
- **调用方式**：在回复下方展示【立即认证】按钮，用户点击后唤起认证表单

### 9.2.3 辅助工具：认证审核进度查询工具

- **工具名称**：【职场认证审核进度查询】
- **工具用途**：查询用户提交的职场认证申请的审核状态

### 9.3.2 交互流程

**步骤一**：意图识别与身份判断

乐享识别到相关意图后，调用【职场人群认证状态查询】工具确认当前用户状态。

**步骤二**：场景分类

**情况一：用户未认证**
- 核心逻辑：引导用户进行职场身份认证，以解锁专属权益
- 示例话术：
  - （直接意图）"您好！联想为在职员工提供专属的职场认证权益。完成认证后，您可以享受产品优惠折扣、专属推荐等权益。点击下方按钮即可快速完成认证。"
  - （产品咨询意图）"您好！针对您的需求，联想有多款适合的产品。检测到您尚未完成职场身份认证，完成认证后可享受专属优惠价格和定制推荐。"
- 交互动作：在回复下方展示【立即认证职场身份】按钮

**情况二：用户已认证（未成为企业会员）**
- 核心逻辑：正常回答 + 权益展示 + 企业会员升级引导
- 示例话术：
  - "您好！您已完成职工身份认证，当前可享受XXXXXX优惠。您还可以一键升级成为企业会员，享受更长期企业积分激励和企业采购服务。"
- 交互动作：在回复下方展示【一键升级成为企业用户】按钮（不需要额外交互，提示认证成功，显示企业会员权益）

**情况三：用户已认证（且是企业会员）**
- 核心逻辑：正常回答 + 职场认证权益 + 企业会员权益
- 示例话术：
  - "您好！您已通过职场认证，当前可享受职场人群优惠，同时享受企业会员采购权益。"
- 交互动作：调用【职场认证权益查询】+【企业会员权益查询】工具

**情况四：用户邮箱待验证（仅邮箱认证场景）**
- 核心逻辑：告知待邮箱验证，告诉用户操作步骤
- 示例话术："您好！您的职场身份认证申请已经提交，请您登录认证邮箱进行验证。"

**情况五：认证失败**
- 核心逻辑：告诉用户审核未通过原因，告知操作步骤
- 示例话术："您好！您的职场身份认证申请未通过系统审核，可以重新提交，推荐通过企业邮箱进行快速认证，也可以通过上传您的工卡、名片或其他带有个人姓名及岗位信息的材料进行职场人员认证。"

### 9.3.3 认证表单唤起与提交

用户点击【立即认证职场身份】按钮后：

1. **唤起认证表单**：弹出认证H5页面（移动端）或页面弹层（PC端）
2. **表单流程**（共4步）：

| 步骤 | 内容 | 说明 |
|------|------|------|
| 第1步 | 认证类型 | 展示企业职工认证（默认选中） |
| 第2步 | 实名认证 | 身份证+手机号验证（已完成则跳过） |
| 第3步 | 企业在职认证 | 企业名称+职位+行业 |
| 第4步 | 在职证明 | 四选一：邮箱/劳动合同/个税APP/其他材料 |

4. **乐享回复**：
   - 邮箱认证："您的职场身份认证申请已成功提交！请您登录邮箱进行验证，验证通过后，您即可享受职工专属权益。"

   - 认证未通过（OCR不通过）："您好！您的职场身份认证申请未通过系统审核，可以重新提交，推荐通过企业邮箱进行快速认证，也可以通过上传您的工卡、名片或其他带有个人姓名及岗位信息的材料进行职场人员认证。"

# 十、后台管理

## 10.1 系统概述

联想在职员工认证后台管理系统是为管理员和客服人员提供的后台工具，主要用于：
- 查看认证数据统计
- 管理已认证员工信息
- **处理OCR审核未通过的申请**（客服接到用户反馈后，在后台修改被驳回的状态）

> **重要说明**：后台审核管理不是日常审核流程，而是为客服/业务人员提供修改被驳回状态的入口。正常认证流程全部由系统自动完成（邮箱验证/OCR审核）。

## 10.2 核心功能

- 在职员工数据概览 — 实时掌握认证数据统计与分析
- 在职员工管理 — 查询和管理通过认证的员工信息
- 认证审核管理 — 查看和处理被OCR驳回的认证申请

## 10.3 菜单结构

```
会员管理
└── 在职员工管理（可展开/收起）
    ├── 在职员工管理
    └── 认证审核管理
```

## 10.4 在职员工数据概览

### 10.4.1 总体统计指标

- 总在职员工数 — 系统中登记的全部员工数
- 已认证员工 — 已完成认证的员工数
- 待认证员工 — 邮箱待验证的员工数
- 本月新增 — 本月新增认证通过的员工数

### 10.4.2 认证方式分布

- 企业邮箱 — 通过企业邮箱验证的员工
- 劳动合同/在职证明 — 上传劳动合同或在职证明的员工
- 个人所得税 — 通过个人所得税APP验证的员工
- 其他材料 — 上传其他证明材料的员工

## 10.5 在职员工管理

### 10.5.1 列表功能

仅显示已通过认证的员工。

### 10.5.2 筛选功能

- 账号/姓名 — 模糊搜索
- 是否实名 — 下拉选择（全部/已认证/未认证）
- 认证方式 — 下拉选择（全部/企业邮箱/劳动合同/个人所得税/其他材料）

### 10.5.3 操作功能

- 查看详情 — 查看员工完整信息和认证材料
- 搜索 — 根据条件搜索员工
- 重置 — 清空所有筛选条件
- 导出 — 导出筛选结果为表格（预留功能）

### 10.5.4 员工详情页

**个人基本信息**：真实姓名、昵称、性别、生日、关联手机号、居住地、注册时间、用户类型

**会员信息**：成为企业会员时间、会员等级、企业等级、激活状态、黑金VIP、注册通路

**企业信息**：所属企业、企业税号、企业邮箱

**职员认证信息**：认证状态、认证方式、认证时间

**认证材料**：验证记录（邮箱/个税等）、上传材料缩略图预览（支持点击查看大图）

## 10.6 认证审核管理

### 10.6.1 审核状态分类

| 状态 | 说明 |
|------|------|
| 邮箱待验证 | 邮箱认证已发送验证码，用户未完成验证 |
| 已通过 | 认证通过（邮箱验证成功/OCR通过/客服手动修改） |
| 已拒绝 | OCR审核未通过 |

备注：
- 已通过和已拒绝状态的申请仅可查看详情
- 已失效的认证（365天有效期到期）不在审核管理中显示，仅在统计数据中保留
- 申请人可重新提交认证申请，作为新申请处理

### 10.6.2 审核操作

在审核详情页可执行以下操作（**仅限"已拒绝"状态，供客服人员使用**）：

**修改为通过**：
- 场景：用户联系客服反馈OCR误判，客服核实后在后台操作
- 操作：点击"通过审核"按钮
- 流程：弹窗确认 → 确认提交
- 结果：申请转为"已通过"状态，申请人获得相应权益
- 有效期：365天

**查看详情模式**：
- 显示所有申请信息和认证材料（只读）
- 显示OCR审核结果和失败原因
- 显示认证方式、企业信息、申请时间等

## 10.7 认证审核详情页

**基础信息区**：申请编号、申请人账号、申请人姓名、关联手机号、LenovoID、所属企业、企业邮箱、申请时间、当前审核状态

**申请信息区**：认证方式（企业邮箱/劳动合同/个人所得税/其他材料）、企业信息（企业名称、行业、职位）

**认证材料区**：申请人上传的完整认证材料、支持预览和下载、显示材料上传时间和文件信息

**OCR审核结果区**：OCR识别结果、通过/不通过原因、提取的关键字段信息

# 十一、业务流程总结

## 11.1 认证申请流程

```
员工发起申请
  ↓
填写个人信息 + 选择认证类型
  ↓
上传认证材料或选择企业邮箱验证
  ↓
【邮箱验证路线】
  发送验证码至企业邮箱
  → 用户回填验证码
  → 验证通过 → 认证成功（即时生效，365天有效期）

【材料审核路线（OCR自动）】
  系统OCR自动审核
  ├→ OCR通过 → 认证成功（即时生效，365天有效期）
  └→ OCR不通过 → 认证失败 → 用户可重新提交或联系客服
       └→ 客服在后台修改状态 → 认证成功

【有效期管理】
  认证有效期：365天
  → 到期自动失效 → 仅在统计数据中保留
  → 用户需重新提交新申请
```

# 十三、常见问题

**Q：认证需要多久？**
A：企业邮箱认证即时通过。其他方式（劳动合同、个税APP、其他材料）通过OCR系统自动审核，通常几分钟内出结果。

**Q：OCR审核没通过怎么办？**
A：可以重新准备更清晰的材料重新提交。推荐使用企业邮箱认证（最快）。也可联系客服，客服核实后可在后台修改审核状态。

**Q：什么是"邮箱待验证"状态？**
A：当用户选择企业邮箱认证时，系统会发送验证码至其企业邮箱。在用户回填验证码通过前，申请处于"邮箱待验证"状态。

**Q：认证过期后如何处理？**
A：认证有效期为365天。过期后原权益自动失效，用户可重新提交认证申请获得新的认证。

**Q：有人工审核环节吗？**
A：没有。所有认证方式均为系统自动审核（邮箱验证码校验 或 OCR识别）。仅当OCR误判时，用户联系客服，客服可在后台修改被驳回的状态。
