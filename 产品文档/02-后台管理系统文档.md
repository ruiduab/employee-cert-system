# 联想员工认证系统 - 后台管理文档

## 系统概述

本系统是一个**员工认证审核与数据管理平台**，用于：
1. **审核管理**：管理和审核员工的实名认证和在职认证申请
2. **数据分析**：统计和分析员工认证数据，了解企业员工分布
3. **精准投放**：基于认证数据，识别不同类型的企业员工，为其投放针对性的商业offer和权益

**系统地址**：http://101.132.158.166:8083/

**数据规模**：
- 总员工数：2,386 人
- 已认证：2,341 人（98.11%）
- 已失效：45 人（1.89%）
- 本月新增：187 人（7.84%）

## 筛选功能详解

### 筛选逻辑

**AND 关系示例**：
```
筛选条件：认证状态 = 已认证 AND 认证类型 = 普通职员 AND 认证方式 = 企业邮箱

结果：2,041 条记录
说明：同时满足这三个条件的员工
```

### 数据一致性

**关键原则**：筛选后的员工列表数量 = 顶部 4 个卡片对应条件的数值

**示例**：
```
点击 "已认证员工" 卡片 (2,341)
→ 筛选条件：认证状态 = 已认证
→ 列表显示数量：2,341 条
→ 分页：第 1 页 / 共 47 页 (2,341 ÷ 50 = 46.82 ≈ 47 页)
```

### 快速筛选（点击卡片）

**点击 "已认证员工" 卡片**：
- 筛选条件：`认证状态 = 已认证`
- 显示员工数：2,341 人

**点击 "已失效员工" 卡片**：
- 筛选条件：`认证状态 = 已失效`
- 显示员工数：45 人

**点击 "普通职员" 卡片**：
- 筛选条件：`认证类型 = 普通职员`
- 显示员工数：1,875 人

**点击 "专业认证" 卡片**：
- 筛选条件：`认证类型 = 专业*` (包括所有专业类型)
- 显示员工数：511 人（186 + 198 + 89 + 38）

## 审核操作

### 认证状态说明

| 状态 | 说明 | 可操作 | 操作内容 |
|------|------|--------|---------|
| **已认证** | 员工已通过实名和在职认证 | ✗ 不可操作 | 仅可查看详情 |
| **已失效** | 员工认证已过期或被撤销 | ✗ 不可操作 | 仅可查看详情，记录失效信息 |
| **待审核** | 员工提交申请，待后台审核 | ✓ 可操作 | 通过 / 拒绝 / 查看详情 |

### 操作按钮状态

**"查看详情" 按钮**：
- 始终可用
- 显示员工的完整认证信息：
  - 基本信息（姓名、身份证号、手机号）
  - 企业信息（企业名称、职位、所属行业）
  - 认证信息（认证类型、认证方式、认证时间）
  - 审核备注

**"审核操作" 按钮**：
- 已认证员工：隐藏
- 已失效员工：隐藏
- 待审核员工：显示为可用按钮，点击弹窗进行审核

**审核弹窗选项**：
- 通过审核 ✓
- 拒绝审核 ✗
- 取消

## 技术实现

### 前端技术栈

- **HTML5**：语义化结构
- **CSS3**：Tailwind CSS + 自定义样式（绿色主调）
- **JavaScript**：原生 JS，无框架
- **数据存储**：浏览器内存
- **数据结构**：JSON 数组，每条记录对应一个 TR 元素

### 数据属性（HTML）

```html
<tr 
  data-status="certified"           <!-- 认证状态：certified/expired -->
  data-cert-type="普通职员"          <!-- 认证类型 -->
  data-cert-method="企业邮箱"        <!-- 认证方式 -->
  data-is-new="yes"                 <!-- 是否本月新增 -->
>
  <!-- 表格单元格 -->
</tr>
```

### 筛选算法

```javascript
// 伪代码
filteredRows = allRows.filter(row => {
    return (statusFilter === 'all' || row.data-status === statusFilter) &&
           (typeFilter === 'all' || row.data-cert-type === typeFilter) &&
           (methodFilter === 'all' || row.data-cert-method === methodFilter) &&
           (newFilter === 'all' || row.data-is-new === 'yes');
});

// 更新表格显示
updatePagination(filteredRows);
```

### 分页算法

```javascript
itemsPerPage = 50;
totalPages = Math.ceil(filteredRows.length / itemsPerPage);
currentPage = 1;

function goToPage(pageNum) {
    startIndex = (pageNum - 1) * itemsPerPage;
    endIndex = startIndex + itemsPerPage;
    displayRows = filteredRows.slice(startIndex, endIndex);
}
```

## 常见操作场景

### 场景1：查找特定员工

1. 使用搜索框输入员工名字/手机号
2. 系统实时搜索并显示相关结果

2. 系统显示所有选择了专业认证的员工
3. 进一步筛选具体专业类型查看详细分布

### 场景4：查看本月新增员工

1. 在筛选条件中选择 "本月新增"
2. 列表显示 187 名本月新增员工

### 场景5：导出员工数据

1. 筛选得到目标员工集合
2. 点击 "导出" 按钮（功能待开发）
3. 下载 Excel 文件

## 故障排查

### 问题1：列表数据不显示

**症状**：页面打开但列表为空

**排查步骤**：
1. 检查浏览器控制台是否有 JavaScript 错误
2. 刷新页面重新加载
3. 检查浏览器缓存（Ctrl+Shift+Delete）
4. 检查 Nginx 日志查看是否有 HTTP 错误

### 问题2：筛选结果不匹配

**症状**：筛选后列表数量与卡片数值不符

**排查步骤**：
1. 检查筛选条件是否正确应用
2. 验证数据属性（data-status、data-cert-type 等）
3. 清除浏览器缓存重新加载
4. 检查 JavaScript 中的筛选逻辑

### 问题3：分页无法跳转

**症状**：点击分页按钮或输入页码无效

**排查步骤**：
1. 检查当前页码是否超出总页数
2. 验证 updatePagination() 函数是否被正确调用
3. 检查浏览器控制台的错误信息
4. 刷新页面重试
