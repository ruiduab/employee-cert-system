# 企业职工认证功能拆分表（研发排期用）

版本：V1.1
日期：2026-04-22
说明：基于所有产品文档及前端原型整理，覆盖官网、乐享智能体、后台三大系统。以V2版本（0418文档）为开发基准。

## 一、官网（联想官网前端认证功能）

| 模块 | 功能 | 功能细分 |
|------|------|---------|
| 认证入口 | 登录态与状态初始化 | 进入流程前检测登录态，未登录跳转登录页 |
| 认证入口 | 登录态与状态初始化 | 页面加载时调用后端API获取认证状态（未认证、基础认证、邮箱待验证、失败、已失效） |
| 认证入口 | 登录态与状态初始化 | 根据认证状态渲染第1步差异化内容（个人用户/企业购用户分支） |
| 第1步-认证类型选择 | 首次认证页面 | 展示认证类型选择与权益介绍，点击下一步触发hasRealname前置检查 |
| 第1步-认证类型选择 | 实名未完成弹窗 | hasRealname=false时弹出提示，提供"继续实名认证"和"返回"按钮 |
| 第2步-实名认证 | 实名表单 | 姓名、身份证号、手机号（预填账号手机）、短信验证码、协议勾选 |
| 第2步-实名认证 | 已完成状态 | hasRealname=true时隐藏表单、显示"实名认证已完成"提示，可直接进入下一步 |
| 第2步-实名认证 | 手机短信验证码 | 调用短信发送接口、60秒冷却倒计时 |
| 第2步-实名认证 | 表单校验 | 姓名2-20汉字、身份证18位、手机号11位、验证码6位、协议必选 |
| 第2步-实名认证 | 实名信息提交 | 提交姓名、身份证、手机、验证码到后端，成功后标记hasRealname=true并跳转第3步 |
| 第3步-企业在职认证 | 企业名称搜索 | 输入2个中文字符后300ms防抖调用天眼查API，下拉展示最多10条结果，必须从列表选择 |
| 第3步-企业在职认证 | 企业信息展示 | 选中后展示企业全称、统一社会信用代码；企业购用户自动带出企业名（只读） |
| 第3步-企业在职认证 | 邮箱后缀查询 | 选中企业后查询是否有企业邮箱后缀，有则绿色提示，无则黄色提示，结果影响第4步选项 |
| 第3步-企业在职认证 | 职位与行业填写 | 职位自动联想（最多10条）、支持自定义、2-12字符限制；行业下拉必选 |
| 第3步-企业在职认证 | 表单校验与提交 | 企业必须从列表选、职位必填2-12字、行业必选，校验通过后提交企业信息进入第4步 |
| 第4步-在职证明提交 | 认证方式选择 | 四选一：企业邮箱、劳动合同或在职证明、个人所得税APP截图、其他材料；无邮箱后缀时隐藏邮箱选项并默认选劳动合同 |
| 第4步-在职证明提交 | 企业邮箱认证 | 邮箱用户名输入、已知后缀自动填充（只读）、无后缀则手动输入后缀、公共邮箱黑名单校验 |
| 第4步-在职证明提交 | 企业邮箱OTP验证 | 发送OTP至企业邮箱（频率限制）、6位验证码回填校验、通过后即时完成认证 |
| 第4步-在职证明提交 | 劳动合同/在职证明上传 | 文件上传（JPG、PNG、PDF，5MB以内），提交后调用OCR自动审核，通过即完成，不通过提示重传或联系客服 |
| 第4步-在职证明提交 | 个人所得税APP截图上传 | 文件上传（JPG、PNG、PDF，5MB以内），调用模板化OCR识别，通过即完成，不通过提示重传 |
| 第4步-在职证明提交 | 其他材料上传 | 多图上传（社保记录、工牌、名片等），调用OCR分类识别，通过即完成，不通过提示重传 |
| 第5步-认证结果页 | 认证成功展示 | 展示成功提示、职场认证礼包权益、"一键升级企业用户"入口（个人用户）、返回首页按钮 |
| 第5步-认证结果页 | 认证失败展示 | 展示OCR未通过原因、"重新提交"按钮、"联系客服"入口 |
| 第5步-认证结果页 | 邮箱待验证展示 | 展示邮箱验证码已发送提示，告知用户去企业邮箱完成验证 |
| 流程控制 | 步骤进度条与导航 | 显示当前步骤（1-4）、已完成步骤标记、允许点击返回已完成步骤 |
| 流程控制 | 返回按钮逻辑 | 第4步→第3步→第2步（显示已完成状态）→第1步，各步骤返回保留已填数据 |
| 流程控制 | 认证有效期管理 | 认证有效期365天，已失效状态展示"重新认证"引导并走相同流程 |
| 前端适配 | 移动端全屏版 | 独立全屏H5页面（393px），蓝色主题（#1677FF），竖向表单、底部按钮布局 |
| 前端适配 | 移动端半弹层版 | 底部弹出90vh，顶部拖拽条、右上角关闭、slideUp动画、半透明遮罩 |
| 前端适配 | PC弹窗版 | 745×456px固定尺寸，iframe嵌入，双列表单，紫红渐变主题 |
| 权益触达 | 认证礼包发放 | 认证成功后触发token、优惠券下发，重复认证不重复发放，对接权益中心 |
| 权益触达 | 企业会员升级引导 | 个人用户认证成功后展示入口，点击触发企业会员注册或升级流程 |
| 第三方集成 | 天眼查API代理 | 后端代理企业搜索接口与邮箱域名查询接口，管理accessToken鉴权 |
| 第三方集成 | 短信验证码服务 | 调用短信服务商发送手机验证码，同一手机5分钟内最多3次限流 |
| 第三方集成 | 邮箱OTP服务 | 调用邮件服务发送企业邮箱OTP，10分钟有效，接口限流 |
| 第三方集成 | OCR自动审核 | 调用视觉Agent对劳动合同提取姓名、公司、职位，交叉校验实名信息，印章检测，置信度>95%通过 |
| 第三方集成 | OCR自动审核 | 个税APP模板化OCR识别；其他材料分类路由+要素提取 |
| 安全与合规 | 敏感信息保护 | 身份证号前端脱敏显示，传输加密，后端AES-256加密存储 |
| 安全与合规 | 防刷限制 | 同一用户24小时内最多提交3次认证申请，IP异常检测，验证码频率限制 |

## 二、智能体（乐享AI对话系统）

| 模块 | 功能 | 功能细分 |
|------|------|---------|
| 意图识别 | 职场认证直接意图 | 识别"职工认证""员工优惠""我想做职场认证"等精准及模糊意图 |
| 意图识别 | 产品咨询+职业身份关联意图 | 识别"我是律师想买笔记本""程序员用什么电脑"等，关联职业与产品推荐 |
| 意图识别 | 认证进度与权益查询意图 | 识别"认证审核了吗""我有什么优惠""职工折扣怎么用"等意图 |
| 意图识别 | 与企业会员意图区分 | "企业采购/批量购买"→企业会员流程；"职工认证/员工优惠"→职场认证流程；模糊时优先职场认证 |
| 工具调用 | 职场认证状态查询工具 | 识别到相关意图后调用，返回认证状态（未登录、未认证、邮箱待验证、成功、失败、已失效）及认证详情 |
| 工具调用 | 审核进度查询工具 | 返回状态（未提交、邮箱待验证、成功、失败）、失败原因、提交时间 |
| 工具调用 | 权益查询工具 | 返回认证类型、权益详情（折扣、适用产品、附加权益）、企业会员权益联动 |
| 工具调用 | 认证唤起工具 | 在回复下方展示"立即认证职场身份"按钮，点击唤起H5弹窗（移动端）或弹层（PC端） |
| 场景A-对话内触发认证 | 未认证用户话术 | 直接意图引导认证+展示权益；产品咨询意图正常回答+认证引导，展示立即认证CTA按钮 |
| 场景A-对话内触发认证 | 已认证未升企业会员话术 | 展示已有职场权益，引导一键升级企业用户，展示升级CTA按钮 |
| 场景A-对话内触发认证 | 已认证已是企业会员话术 | 调用权益查询工具，展示职场认证权益与企业会员权益 |
| 场景A-对话内触发认证 | 邮箱待验证话术 | 告知验证码已发送，提示用户登录企业邮箱完成验证 |
| 场景A-对话内触发认证 | 认证失败话术 | 说明失败原因，引导重新提交，推荐邮箱快速认证，展示"重新认证"按钮 |
| 场景A-对话内触发认证 | 表单提交后自动回调 | 表单提交成功后前端向乐享发送"提交职场身份认证"query，乐享根据OCR或邮箱状态返回对应话术 |
| 场景B-活动页触发认证 | 活动入口联动 | 活动页展示"职工专属价"标签，未认证用户点击展示引导文案和立即认证按钮，点击唤起乐享侧边栏 |
| 场景B-活动页触发认证 | 已认证用户直接展示 | 已认证用户点击后直接展示职工专属价格 |
| 场景C-认证进度查询 | 各状态回复逻辑 | 未提交→引导认证；邮箱待验证→提供验证操作指引；成功→展示权益；失败→展示原因+重新认证按钮 |
| 场景D-权益查询 | 权益展示 | 已认证用户调用状态查询+权益查询展示完整权益卡片；未认证用户介绍权益+立即认证按钮 |
| 知识库与工具链路 | 认证FAQ知识配置 | 配置认证材料说明、审核时长、有效期、失败处理、权益介绍等FAQ |
| 知识库与工具链路 | 工具调用链路与容错 | 先查状态→再决定后续工具调用；工具失败时返回"网络繁忙，请稍后重试"，不出现无响应 |

## 三、后台（在职员工认证后台管理系统）

| 模块 | 功能 | 功能细分 |
|------|------|---------|
| 数据概览 | 核心统计指标 | 总在职员工数、已认证员工数、邮箱待验证数、本月新增数实时统计卡片 |
| 数据概览 | 认证方式分布 | 按企业邮箱、劳动合同、个人所得税、其他材料统计员工数量与占比，可视化图表展示 |
| 在职员工管理 | 员工列表 | 字段：账号（脱敏）、真实姓名、LenovoID、手机号、是否实名、所属企业、信用代码、认证方式、认证时间；分页25条 |
| 在职员工管理 | 列表筛选 | 按账号、姓名模糊搜索；按是否实名、认证方式筛选；多条件AND逻辑，筛选后回第1页 |
| 在职员工管理 | 数据导出 | 导出当前筛选结果为Excel，支持自定义导出字段，大数据量异步处理 |
| 在职员工管理 | 员工详情查看 | 个人信息、会员信息、企业信息、职员认证信息完整展示，认证材料缩略图预览与下载 |
| 认证审核管理 | 审核列表 | 三态标签页（邮箱待验证、已通过、已拒绝），字段：申请编号、申请人、认证方式、申请时间、审核人 |
| 认证审核管理 | 审核列表筛选 | 按申请编号、账号或姓名搜索；按认证方式、申请时间范围筛选 |
| 认证审核管理 | 审核详情页 | 基础信息、申请信息（认证方式、企业、行业、职位）、认证材料预览下载、OCR审核结果（识别字段、通过或失败原因） |
| 认证审核管理 | 通过审核操作 | 仅"邮箱待验证"或"已拒绝"状态可操作，确认弹窗二次确认后状态转"已通过"，申请人获得权益（365天） |
| 认证审核管理 | 拒绝申请操作 | 仅"邮箱待验证"状态可操作，必须填写拒绝原因，确认后状态转"已拒绝"并通知申请人 |
| 认证审核管理 | 客服修改驳回状态 | 仅"已拒绝"状态可操作（OCR误判修正），确认弹窗二次确认，操作记录留痕 |
| 认证审核管理 | 已通过、已拒绝详情只读 | 展示全部申请信息、材料、审核人、审核时间；已通过含有效期；已拒绝含拒绝原因；不显示操作按钮 |
| 权限与角色 | 角色权限管理 | 角色：超级管理员、审核主管、审核员、数据分析员、查看者；审核批准或拒绝操作仅审核主管及以上可执行 |
| 权限与角色 | 敏感信息脱敏 | 身份证号、手机号默认脱敏，需权限才能查看完整信息，审核意见仅审核人及以上可见 |
| 有效期与重认证 | 到期自动失效 | 认证365天后自动标记已失效，失效记录不在审核管理中显示，保留在统计数据中 |
| 有效期与重认证 | 重新认证处理 | 已失效用户重新提交作为新申请处理，新有效期重新计算365天 |
| 通知与系统设置 | 审核结果通知 | 审核通过或拒绝后通知申请人（邮件、站内信），通知开关可配置 |
| 通知与系统设置 | 规则参数配置 | 有效期（365天）、单文件大小限制（5MB）、允许格式（JPG、PNG、PDF）、在职证明有效期要求（6个月内） |
| 后端API | 认证状态查询、实名提交、企业搜索 | GET认证状态接口；POST实名认证接口（接收身份证、姓名、手机、验证码，调用三方验证）；GET企业搜索代理接口（返回全称、信用代码、邮箱域名、行业，最多10条） |
| 后端API | 在职信息提交、邮箱验证 | POST在职信息接口（保存企业、职位、行业，返回cert_id）；邮箱OTP发送接口（校验黑名单、发送OTP、限流）；邮箱OTP校验接口（核对验证码、更新状态、触发权益发放） |
| 后端API | 材料上传、OCR回调、状态推送 | POST材料上传接口（multipart上传至OSS、触发OCR异步审核）；OCR结果回调接口（更新状态、存储失败原因、触发通知）；前端轮询或WebSocket推送认证结果 |
| 后端API | 审核操作、权益发放 | POST审核通过、拒绝接口（记录审核人、时间、意见）；权益查询接口；认证成功后礼包（token、优惠券）下发，重复认证不重复发放 |
| 数据库设计 | 核心数据表 | user_certification（认证状态、方式、实名信息、企业信息、材料URL、有效期）；certification_material（文件URL、类型、OCR状态、审核备注）；review_record（审核人、时间、操作、意见） |
| 数据库设计 | 企业库与缓存 | company_whitelist表（企业全称、信用代码、邮箱域名、行业分类，支持模糊搜索），Redis缓存（TTL 7天） |
| 日志与审计 | 操作日志 | 所有审核操作记录（操作人、时间、内容），敏感操作二次确认，日志查询功能 |

## 附：版本说明

| 版本 | 主要差异 |
|------|---------|
| V1（0410文档）| 含L2专业职称认证（法律、工业设计、媒体设计、编程）体系；含人工审核流程；专项认证有独立权益和机型推荐 |
| V2（0418文档）| 移除专业职称分层；统一为单一"职场人群认证"；全自动OCR审核，无人工审核；后台仅供客服修改OCR误判 |

研发排期建议以V2版本（0418文档）为开发基准。
