## 概述

系统支持用户的**多次认证**和**升级认证**，允许已认证用户升级为专业职称认证，或更新基本的企业和职位信息。

### 核心特性
- ✅ 首次认证：用户首次进行完整的企业认证
- ✅ 升级认证：已基础认证的用户升级为专业职称认证
- ✅ 更新认证：已认证用户更新企业或职位信息
- ✅ 历史记录：系统记录每次认证的完整历史

---

## 用户认证状态

系统根据用户的认证历史，分为三个主要状态：

### 状态1：未认证 (not-certified)
- **适用**：首次使用系统的用户
- **特征**：无任何认证历史
- **可执行操作**：
  - 企业职工认证
  - 专业职称认证
- **Step 1显示**：两个认证选项卡

### 状态2：已基础认证 (basic-certified)
- **适用**：已通过邮箱/合同/税单认证，但非定义专业职称的用户（如销售）
- **特征**：有至少一次企业认证，但未进行专业职称认证
- **可执行操作**：
  - ✨ 升级为专业职称认证
  - 更新在职认证信息
- **Step 1显示**：两个新的入口（升级 + 更新）

### 状态3：已完成认证 (specialist-certified)
- **适用**：已完成专业职称认证的用户
- **特征**：有专业职称认证记录
- **可执行操作**：
  - 更新在职认证信息
- **Step 1显示**：一个入口（更新）

## 三种认证流程

### 流程A：首次认证 (First-time Certification)

**用户状态**：未认证 (not-certified)

**步骤**：
```
Step 1: 选择认证类型
  ↓
Step 2: 实名认证（身份证 + 手机 + 验证码）
  ↓
Step 3: 企业在职信息（企业 + 职位 + 行业）
  ↓
Step 4: 认证材料（邮箱 / 合同 / 税单 / 其他）
  ↓
Step 5: 成功页面
```

**关键特征**：
- 用户走完整的4个认证步骤
- 需要填写所有字段
- 后台记录为 `认证次数 = 1`、`是否升级 = false`

**用户提交时传递数据**：
```javascript
{
    certNumber: 1,           // 第1次认证
    isUpgrade: false,        // 不是升级
    certType: 'basic' or 'specialist_xxx',
    company: '联想（北京）有限公司',
    position: '产品经理',
    industry: '互联网/信息技术',
    authMethod: 'email',
    email: 'user@lenovo.com',
    // ...
}
```

### 流程B：升级认证 (Upgrade Certification)

**用户状态**：已基础认证 (basic-certified)

**用户操作**：在Step 1 点击"升级：专业职称认证"入口

**步骤**：
```
Step 1: 显示升级认证入口 ← [用户点击"升级"]
  ↓
[系统检测：hasRealname = true，跳过Step 2]
  ↓
Step 3: 选择专业职称类型（仅显示专业选择，隐藏企业/职位字段）
  ↓
Step 4: 职称材料补充（仅显示"其他材料补充"选项）
  ↓
Step 5: 成功页面
```

**关键特征**：
- ⏭️ **跳过Step 1/2**：用户已有实名认证
- 🎯 **重点是补充职称证明**：而非重新验证企业身份
- 📝 **专业职称选择**：选择4个专业之一（法律、工业设计、媒体设计、编程开发）
- 📎 **仅需上传材料**：设计作品集、律师证、技术证书等职称相关文件
- 🏷️ **后台记录**：`认证次数 = 2+`、`是否升级 = true`、`认证类型 = 专业职称`

**Step 4 的提示文案示例**：
```
专业职称认证资料上传

您已完成企业在职认证，现在需要补充专业资格证明。

✓ 设计师 → 设计相关的项目文件、作品集、资格证书等截图
✓ 律师 → 律师证、执业证、法律相关文件截图
✓ 编程开发 → 技术认证证书、代码贡献证明、项目经验等截图
✓ 工业设计 → 设计专利、获奖证书、设计作品等截图

📌 建议：上传清晰的身份证件或资格证书照片
```

**用户提交时传递数据**：
```javascript
{
    certNumber: 2,           // 第2次认证（或更高）
    isUpgrade: true,         // 是升级认证
    certType: 'specialist_media',  // 升级为哪个专业
    authMethod: 'other',     // 仅允许其他材料
    filePath: '/uploads/design_portfolio_2026.pdf',
    fileName: 'design_portfolio.pdf',
    // 注意：不包含企业、职位、行业等信息（沿用第1次认证的数据）
}
```

### 流程C：更新认证 (Update Certification)

**用户状态**：已基础认证或已完成认证 (basic-certified 或 specialist-certified)

**用户操作**：在Step 1 点击"更新在职认证信息"入口

**步骤**：
```
Step 1: 显示更新入口 ← [用户点击"更新"]
  ↓
[系统检测：hasRealname = true，跳过Step 2]
  ↓
Step 3: 企业在职信息（显示上一次认证的信息，可编辑）
  ↓
Step 4: 认证材料（显示所有认证方式，与首次认证相同）
  ↓
Step 5: 成功页面
```

**关键特征**：
- ⏭️ **跳过Step 1/2**：用户已有实名认证
- 📝 **预填上一次数据**：企业、职位、行业信息来自上一次认证
- ✏️ **完全可编辑**：用户可修改任何字段
- 🔄 **同样的认证方式选择**：邮箱、合同、税单、其他都可选
- 🏷️ **后台记录**：`认证次数 = 2+`、`是否升级 = false`、`认证类型 = 企业职工`

**示例场景**：
- 用户从联想离职加入阿里，需要更新企业信息
- 用户职位晋升，需要更新职位信息
- 用户之前用邮箱认证，现在想用合同认证

**用户提交时传递数据**：
```javascript
{
    certNumber: 2,           // 第2次认证（或更高）
    isUpgrade: false,        // 不是升级，是更新
    certType: 'basic',       // 保持企业职工认证类型
    company: '阿里巴巴（杭州）有限公司',  // 更新的企业
    position: '技术负责人',  // 更新的职位
    industry: '互联网/信息技术',  // 更新的行业
    authMethod: 'contract',  // 这次选择合同认证
    filePath: '/uploads/labor_contract_2026.pdf',
    fileName: 'labor_contract.pdf',
}
```

## 前端实现要点

### 1. 检测用户状态

```javascript
// 页面加载时检测用户状态
async function initCertification() {
    try {
        // 从后端API获取用户的认证状态
        const response = await fetch('/api/user/cert-status');
        const data = await response.json();
        
        userStatus = data.status;  // 'not-certified' | 'basic-certified' | 'specialist-certified'
        userCertCount = data.certCount;  // 用户已认证的次数
        lastCertData = data.lastCertData;  // 上一次认证的数据（用于预填）
        
        renderStep1();
    } catch (error) {
        // 失败回退到未认证状态
        userStatus = 'not-certified';
        renderStep1();
    }
}
```

### 2. 动态渲染Step 1

```javascript
function renderStep1() {
    const step1 = document.getElementById('step1');
    
    if (userStatus === 'not-certified') {
        // 首次认证：显示企业职工认证 + 专业职称认证选项卡
        step1.innerHTML = `
            <h2>选择认证方式</h2>
            <div class="tabs">
                <button onclick="selectType(this, 'enterprise')">
                    企业职工认证
                </button>
                <button onclick="selectType(this, 'specialist')">
                    专业职称认证
                </button>
            </div>
        `;
    } else {
        // 已认证：显示升级和更新入口
        step1.innerHTML = `
            <h2>继续认证</h2>
            <div class="options">
                ${userStatus === 'basic-certified' ? `
                    <div class="option" onclick="startUpgradeFlow()">
                        <h3>🎯 升级：专业职称认证</h3>
                        <p>从基础认证升级为专业职称认证，获得专业身份和特权</p>
                    </div>
                ` : ''}
                <div class="option" onclick="startUpdateFlow()">
                    <h3>🔄 更新在职认证信息</h3>
                    <p>更新您的企业、职位、行业等信息</p>
                </div>
            </div>
        `;
    }
}
```

### 3. 处理流程跳转

```javascript
function startUpgradeFlow() {
    // 升级流程：跳过Step 1/2
    hasRealname = true;
    selectedType = 'specialist';
    currentStep = 3;  // 跳到Step 3
    flowType = 'upgrade';  // 标记为升级流程
    showStep(3);
}

function startUpdateFlow() {
    // 更新流程：跳过Step 1/2
    hasRealname = true;
    selectedType = 'enterprise';  // 保持企业职工认证
    currentStep = 3;  // 跳到Step 3
    flowType = 'update';  // 标记为更新流程
    
    // 预填上一次的数据
    if (lastCertData) {
        document.getElementById('company').value = lastCertData.company;
        document.getElementById('position').value = lastCertData.position;
        document.getElementById('industry').value = lastCertData.industry;
    }
    
    showStep(3);
}
```

### 4. Step 3 中区分流程

```javascript
function showStep(step) {
    if (step === 3) {
        if (flowType === 'upgrade') {
            // 升级认证：隐藏企业/职位字段，显示专业选择
            document.getElementById('company-section').classList.add('hidden');
            document.getElementById('position-section').classList.add('hidden');
            document.getElementById('specialty-selection').classList.remove('hidden');
        } else {
            // 首次或更新认证：显示企业/职位字段
            document.getElementById('company-section').classList.remove('hidden');
            document.getElementById('position-section').classList.remove('hidden');
            document.getElementById('specialty-selection').classList.add('hidden');
        }
    }
}
```

### 5. Step 4 中区分认证方式

```javascript
function showStep(step) {
    if (step === 4) {
        if (flowType === 'upgrade') {
            // 升级认证：仅显示"其他材料补充"
            document.getElementById('email-option').classList.add('hidden');
            document.getElementById('contract-option').classList.add('hidden');
            document.getElementById('tax-option').classList.add('hidden');
            
            // 自动选中"其他材料"
            selectMethod(
                document.getElementById('other-option'),
                'other'
            );
            
            // 显示职称认证提示
            showSpecialistTips(selectedSpecialty);
        } else {
            // 首次或更新认证：显示所有选项
            document.getElementById('email-option').classList.remove('hidden');
            document.getElementById('contract-option').classList.remove('hidden');
            document.getElementById('tax-option').classList.remove('hidden');
        }
    }
}
```

### 6. 数据提交

```javascript
function submitCertification() {
    const certData = {
        // 流程信息
        flowType: flowType,  // 'first' | 'upgrade' | 'update'
        
        // 自动计算的字段
        certNumber: calculateCertNumber(),  // 认证次数
        isUpgrade: flowType === 'upgrade',
        
        // 认证类型
        certType: selectedType === 'specialist' 
            ? `specialist_${selectedSpecialty}`
            : 'basic',
        
        // 根据流程类型包含不同的字段
        ...(flowType !== 'upgrade' && {
            // 升级认证不需要企业信息
            company: document.getElementById('company').value,
            position: document.getElementById('position').value,
            industry: document.getElementById('industry').value,
        }),
        
        // 认证方式
        authMethod: document.querySelector('input[name="method"]:checked').value,
        
        // 材料信息
        filePath: uploadedFilePath,
        fileName: uploadedFileName,
    };
    
    // 提交到后端
    fetch('/api/auth/submit', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(certData)
    })
    .then(res => res.json())
    .then(data => {
        if (data.success) {
            // 成功：进入Step 5
            currentStep = 5;
            showStep(5);
            
            // 更新本地存储的用户状态
            localStorage.setItem('userCertStatus', data.newStatus);
        }
    });
}
```

## 数据库设计

### 核心表结构

**user_cert_status 表**：记录用户当前的认证状态
```sql
uid | status | cert_count | current_cert_type | last_cert_time
```

**certification_history 表**：记录每次认证的完整信息
```sql
id | uid | cert_number | cert_type | is_upgrade | company | position | 
auth_method | file_path | review_status | reviewer_id | submitted_at | reviewed_at
```

**示例查询**：
```sql
-- 查询用户张三的所有认证记录
SELECT * FROM certification_history 
WHERE uid = 10001 
ORDER BY cert_number DESC;

-- 查询所有待审核的升级认证
SELECT * FROM certification_history 
WHERE is_upgrade = true AND review_status = 'pending' 
ORDER BY submitted_at DESC;

-- 查询本周提交的更新认证
SELECT * FROM certification_history 
WHERE is_upgrade = false AND submitted_at >= DATE_SUB(NOW(), INTERVAL 7 DAY) 
ORDER BY submitted_at DESC;
```
