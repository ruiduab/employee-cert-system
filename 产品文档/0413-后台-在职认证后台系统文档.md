## 一、系统概述

### 1.1 系统目的
联想在职员工认证后台管理系统是为管理员和审核人员提供的专业后台工具，用于管理和审核员工的在职身份认证申请，确保认证流程的规范化和透明化。

### 1.2 核心功能
- **在职员工数据概览** - 实时掌握认证数据统计与分析
- **在职员工管理** - 查询和管理通过认证的员工信息
- **认证审核管理** - 审核和处理员工认证申请

### 1.3 系统定位
- **用户角色**: 管理员、审核主管、审核员、数据分析员
- **应用场景**: 企业员工身份认证管理、权益分配、数据统计
- **部署环境**: Web应用，支持浏览器访问

## 三、功能模块详解

### 3.1 在职员工数据概览

#### 3.1.1 总体统计指标
展示实时的认证统计数据：
- **总在职员工数** - 系统中登记的全部员工数
- **已认证员工** - 已完成认证审核的员工数
- **待认证员工** - 尚未认证或认证待审核的员工数
- **本月新增** - 本月新增认证通过的员工数

#### 3.1.2 职员认证状态分布
统计不同认证类型的员工数量：
- **普通职员** - 完成基础职员认证的员工
- **专业：法律** - 法律专业认证的员工
- **专业：工业设计** - 工业设计专业认证的员工
- **专业：媒体设计** - 媒体设计专业认证的员工
- **专业：编程开发** - 编程开发专业认证的员工

#### 3.1.3 认证方式分布
展示员工采用的认证方式统计：
- **企业邮箱** - 通过企业邮箱验证的员工
- **劳动合同/在职证明** - 上传劳动合同或在职证明的员工
- **个人所得税** - 通过个人所得税APP验证的员工
- **其他材料** - 上传其他证明材料的员工

### 3.3 认证审核管理

#### 3.3.1 审核状态分类

系统在审核管理中维护以下3个状态：

| 状态 | 说明 | 数量示例 | 操作权限 | 操作内容 |
|-----|------|--------|--------|--------|
| 待审核/邮箱待激活 | 刚提交的申请或邮箱已发送验证码但待用户激活 | 51 | 可审核 | ✓ 通过 / ✗ 拒绝 |
| 已通过 | 认证审核通过，授予相应职员身份，有效期365天 | 2,341 | 仅查看 | 查看详情 |
| 已拒绝 | 申请被驳回，申请人可重新申请 | 156 | 仅查看 | 查看详情 |

**备注**：
- 已通过和已拒绝状态的申请不可再次审核，仅可查看详情
- 已失效的认证（365天有效期到期）不在审核管理中显示，但系统记录追踪已失效信息
- 已失效的记录仅保存在统计数据中，用于长期数据分析和趋势报表
- 申请人可以重新提交认证申请，作为新申请处理

#### 3.3.2 审核列表

**表格字段**：

| 字段名 | 说明 | 示例值 |
|-------|------|-------|
| 申请编号 | 认证申请的唯一编号 | APP20240415001 |
| 申请人 | 提交申请的员工信息 | 王五 (user005) |
| 认证方式 | 员工选择的认证方式 | 劳动合同 |
| 认证身份 | 申请的职员认证类型 | 普通职员/专业：xxx |
| 申请时间 | 申请提交的时间 | 2024-04-15 14:30 |
| 审核人 | 负责审核的人员 | 李明 / - |
| 操作 | 审核操作 | 审核 |

#### 3.3.3 审核操作

在审核详情页可执行以下操作（仅限"待审核/邮箱待激活"状态）：

**1. 通过审核 (✓ 批准)**
- 操作：点击"通过审核"按钮
- 流程：弹窗确认 → 选择授予的职员身份 → 确认提交
- 结果：申请转移到"已通过"状态，申请人获得相应权益
- 有效期：365天，到期后自动失效（不再审核管理中显示）

**2. 拒绝申请 (✗ 拒绝)**
- 操作：点击"拒绝申请"按钮
- 流程：输入拒绝原因 → 确认提交
- 结果：申请转移到"已拒绝"状态，申请人被通知
- 申请人可重新准备材料后重新申请

#### 3.3.4 材料审核

**审核材料显示**（仅在"待审核/邮箱待激活"状态下）：
- 显示申请人上传的所有认证材料
- 材料类型支持：图片（jpg、png）、PDF、视频
- 缩略图展示，点击可查看大图或完整内容

**材料验证**：
- 检查材料完整性（清晰度、无遮挡）
- 验证信息一致性（与申请表单是否匹配）
- 审查有效期（如在职证明需6个月内）

#### 3.3.5 认证审核详情页

**页面展示内容**：

**基础信息区**：
- 申请编号、申请人账号、申请人姓名、关联手机号
- LenovoID、所属企业、企业邮箱
- 申请时间、当前审核状态

**申请信息区**：
- 职员认证类型（普通职员/专业认证类型）
- 认证方式（企业邮箱/劳动合同/个人所得税/其他材料）
- 企业信息（企业名称、行业、职位）

**认证材料区**：
- 申请人上传的完整认证材料
- 支持预览和下载
- 显示材料上传时间和文件信息

**审核操作区**（仅"待审核/邮箱待激活"状态）：

**通过审核操作**：
- 按钮：`✓ 通过审核`
- 点击后弹出确认窗口
- 选择授予的职员身份：
  - 普通职员
  - 专业：法律
  - 专业：工业设计
  - 专业：媒体设计
  - 专业：编程开发
- 可选填写备注信息
- 确认提交后，申请状态转为"已通过"
- 申请人获得相应权益，认证有效期为365天

**拒绝申请操作**：
- 按钮：`✗ 拒绝申请`
- 点击后弹出拒绝窗口
- 必填：拒绝原因（文本框，支持多行输入）
- 可选：补充说明或建议
- 确认提交后，申请状态转为"已拒绝"
- 申请人将收到通知，可重新提交申请

**查看详情模式**（仅"已通过"和"已拒绝"状态）：

**已通过状态下的查看详情**：
- 显示所有申请信息和认证材料（只读）
- 显示通过的职员身份和授权信息
- 显示审核人和审核时间
- 显示认证有效期和失效时间
- 不显示任何操作按钮
- 支持导出或打印认证记录

**已拒绝状态下的查看详情**：
- 显示所有申请信息和认证材料（只读）
- 显示拒绝原因和审核意见
- 显示审核人和审核时间
- 显示一个"重新申请"提示或链接
- 不显示任何审核操作按钮
- 支持导出拒绝记录供申请人参考

**权限说明**：
| 审核状态 | 可执行操作 | 说明 |
|---------|---------|------|
| 待审核/邮箱待激活 | 通过审核、拒绝申请 | 审核人可进行审核决策 |
| 已通过 | 查看详情 | 仅可查看认证信息，不可更改 |
| 已拒绝 | 查看详情 | 仅可查看拒绝原因，申请人可重新申请 |

## 五、业务流程

### 5.1 认证申请流程

```
员工发起申请
  ↓
填写个人信息 + 选择认证类型
  ↓
上传认证材料或选择企业邮箱验证
  ↓
后台接收申请 → 状态：待审核/邮箱待激活
  ↓
【邮箱验证路线】
  发送验证码至企业邮箱
    ↓
  用户点击验证链接激活
    ↓
  邮箱验证通过 → 状态：已通过 → 授予权益(365天有效)
  
【材料审核路线】
  审核员审核上传的材料
    ├→ 通过审核 → 状态：已通过 → 授予权益(365天有效)
    └→ 拒绝申请 → 状态：已拒绝 → 通知申请人
  
【有效期管理】
  认证有效期：365天
    ↓
  有效期到期 → 自动失效 → 仅在统计数据中保留
  
  重新认证：用户需重新提交新申请
```

### 5.2 在职员工管理流程

```
管理员进入在职员工管理页
  ↓
[可选] 根据条件筛选查询
  ↓
查看员工列表（只显示已通过认证的员工）
  ↓
点击"查看详情"
  ↓
查看完整的员工信息
  ├─ 个人基本信息
  ├─ 会员信息
  ├─ 企业信息
  ├─ 职员认证信息
  └─ 认证材料（可预览）
```

### 5.3 数据统计流程

```
系统自动汇总认证数据（实时/定时）
  ↓
统计维度：
  ├─ 认证状态分布（普通/法律/工业设计/媒体设计/编程开发）
  ├─ 认证方式分布（邮箱/合同/个税/其他）
  └─ 总体指标（总数/已认证/待认证/本月新增）
  ↓
在数据概览页展示
  ↓
支持下钻分析（点击卡片可深入查看）
```

## 六、字段规范

### 6.1 账号规范
- **格式**: 脱敏显示，保留首尾，中间用****替代
- **示例**: 170****2837
- **长度**: 11位手机号

### 6.2 企业信息规范
- **企业名称**: 完整企业法人名称
  - 示例：联想（北京）有限公司、联想控股股份有限公司
- **企业税号**: 统一社会信用代码，18位
  - 格式：数字+字母
  - 示例：91110105MA00000000
- **企业邮箱**: 标准邮箱格式
  - 示例：zhangsan@lenovo.com

### 6.3 时间格式规范
- **日期**: YYYY-MM-DD
- **日期时间**: YYYY-MM-DD HH:MM:SS
- **示例**: 2024-01-15 14:30:00

### 6.4 职员认证类型说明

| 认证类型 | 说明 | 权益 | 行业限制 | 职位要求 |
|--------|------|------|--------|---------|
| 普通职员 | 基础的在职员工认证 | 全系列产品95折 | 无 | 无 |
| 法律 | 法律领域专业人士 | 95折+定制优惠 | 仅法律 | 律师、法务等 |
| 工业设计 | 工业设计专业人士 | 95折+定制优惠 | 工业设计、制造业、消费电子 | 工业设计师等 |
| 媒体设计 | 媒体设计专业人士 | 95折+定制优惠 | 设计、媒体、广告、创意产业 | UI/平面设计师等 |
| 编程开发 | 编程开发专业人士 | 95折+定制优惠 | 互联网/信息技术、软件开发、云计算 | 工程师、开发者等 |

### 6.5 认证方式说明

| 认证方式 | 说明 | 所需材料 | 审核流程 |
|--------|------|--------|--------|
| 企业邮箱 | 通过企业邮箱接收验证码 | 企业邮箱地址 + 验证码 | 自动验证 |
| 劳动合同/在职证明 | 上传劳动合同或企业开具的在职证明 | JPG/PNG/PDF，单个不超过5MB | 人工审核 |
| 个人所得税 | 上传个人所得税APP的在职受雇信息截图/录屏 | 视频或截图 | 人工审核 |
| 其他材料 | 上传社保缴费、职称证书等其他证明 | JPG/PNG/PDF，可多项组合 | 人工审核 |

### 7.2 功能权限矩阵

## 九、常见问题

### 9.1 为什么员工列表中看不到某个员工？
答：列表中仅显示已通过认证审核的员工。如果员工的申请还在审核中，请在"认证审核管理"中查看。

### 9.2 如何查看申请人上传的认证材料？
答：在员工详情页面的"认证材料"部分，可看到所有上传的材料缩略图，点击可查看大图。

### 9.3 认证申请被拒绝后，申请人可以重新申请吗？
答：可以。申请人可以针对拒绝原因重新准备材料，重新提交认证申请。

### 9.4 企业邮箱认证和其他方式认证有什么区别？
答：企业邮箱认证是自动验证，速度最快。发送验证码后，用户需要点击激活。其他方式需要人工审核申请人上传的材料。

### 9.5 职员身份的有效期是多久？
答：职员身份有效期为365天（一年）。有效期到期后，状态自动转为"已失效"，用户需要重新申请认证以继续享受权益。

### 9.6 什么是"邮箱待激活"状态？
答：当申请人选择企业邮箱认证时，系统会发送验证码至其企业邮箱。在用户点击验证链接激活前，申请处于"邮箱待激活"状态。

### 9.7 已失效的认证记录在哪里查看？
答：已失效的认证记录不在审核管理中显示。仅保存在统计数据中供长期分析和趋势报表。用户需要重新认证时，应重新提交申请，作为新申请处理。

### 9.8 用户认证过期后如何处理？
答：用户认证过期后，原权益自动失效。用户可以重新提交认证申请获得新的认证，新认证有效期同样为365天。

## 十、后续规划
