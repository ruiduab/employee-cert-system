# 联想员工认证系统 - 系统架构与集成指南

## 系统概述

联想员工认证系统是一个**员工身份验证和精准投放平台**，用于识别企业用户身份并提供精准化的商业服务。系统包括三个核心部分：

- **前端**：员工认证申请系统（5步流程，包括成功页）
  - 用户完成实名认证、企业在职认证和材料提交
  - 支持企业职工认证和4种专业职称认证

- **后台**：认证审核与数据管理系统（2,386员工数据管理）
  - 管理员审核员工的认证申请
  - 分析员工认证数据，支持精准offer投放
  - 支持多维度筛选和搜索

- **后端**：认证服务 API（待开发）
  - 处理认证申请、验证逻辑、数据存储
  - 提供短信/邮件验证码服务
  - 支持天眼查企业数据查询

## 前后端数据流向

### 场景1：员工提交认证申请（前端 → 后端）

```
前端界面
  ↓
Step 1: 选择认证类型
  ├─ payload: { certType: 'basic' | 'specialist', specialty: 'lawyer' | ... }
  ↓
Step 2: 填写实名信息
  ├─ POST /api/auth/realname
  ├─ payload: { 
  │    name: '张三',
  │    idcard: '110101199003071234',
  │    phone: '13812345678',
  │    code: '123456'
  │  }
  ├─ response: { success: true, uid: 10001 }
  ↓
Step 3: 填写企业信息
  ├─ POST /api/auth/employee
  ├─ payload: {
  │    company: '联想（北京）有限公司',
  │    position: '产品经理',
  │    industry: '互联网/信息技术'
  │  }
  ├─ response: { success: true }
  ↓
Step 4: 选择认证方式并提交
  ├─ POST /api/auth/material
  ├─ payload: {
  │    method: 'email' | 'contract' | 'tax' | 'other',
  │    email: 'zhangsan@lenovo.com',  // 仅 email 方式
  │    file: <binary>  // 其他方式
  │  }
  ├─ response: { success: true, status: 'pending_review' }
  ↓
数据库存储
  ├─ users 表新增一条记录
  ├─ realname_auth 表存储认证记录
  ├─ employee_auth 表存储在职认证
  └─ material_auth 表存储材料

后台系统自动更新显示
  ├─ 员工列表中新增这条记录
  ├─ 状态标记为 "待审核"
  └─ 通知审核人员
```

### 场景2：审核员工申请（后台 → 后端 → 前端）

```
后台审核员工
  ↓
点击 "审核操作"
  ├─ 弹窗选择 "通过" 或 "拒绝"
  ├─ POST /api/employee/review
  ├─ payload: {
  │    uid: 10001,
  │    action: 'approve' | 'reject',
  │    remark: '审核意见'
  │  }
  ↓
后端处理
  ├─ 更新 users 表状态为 'approved'
  ├─ 写入 audit_log 审核记录
  ├─ 发送通知邮件给申请人
  └─ response: { success: true }
  ↓
后台系统自动更新
  ├─ 员工记录的状态变为 "已认证"
  ├─ 审核操作按钮隐藏
  ├─ 显示审核时间
  └─ 通知前端（若有订阅）

员工前端系统
  ├─ 轮询检查认证状态
  ├─ 显示 "认证已通过" 或 "认证被拒绝"
  └─ 对应显示成功或失败页面
```

### 场景3：后台筛选查询（后端查询数据库）

```
后台点击统计卡片或调整筛选条件
  ↓
触发 filterByCard() 或 filter() 函数
  ├─ 前端从 HTML 中解析当前筛选条件
  ├─ 本地 JavaScript 过滤（当前 demo 版本）
  ↓
（后续集成后）
后端查询数据库
  ├─ GET /api/employee/list?status=certified&type=basic&method=email
  ├─ 执行 SQL：
  │  SELECT * FROM employees 
  │  WHERE status='certified' AND type='basic' AND method='email'
  │  LIMIT 50 OFFSET (page-1)*50
  ↓
数据库返回结果
  ├─ 总记录数：2,041
  ├─ 当前页数据：50 条
  └─ response: {
       total: 2041,
       page: 1,
       pageSize: 50,
       data: [...]
     }
  ↓
前端渲染列表
  ├─ 更新分页条显示："第 1 页 / 共 41 页"
  ├─ 渲染表格行
  └─ 更新统计卡片数值（2,041）
```

## 数据库表设计（待实现）

### users 表（用户/员工基本信息）
```sql
CREATE TABLE users (
    uid INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    idcard VARCHAR(18) UNIQUE NOT NULL,
    phone VARCHAR(11) UNIQUE NOT NULL,
    email VARCHAR(100),
    cert_type VARCHAR(20),  -- 'basic' | 'specialist'
    specialty VARCHAR(20),  -- 'lawyer' | 'industrial_design' | 'media_design' | 'developer'
    status VARCHAR(20) DEFAULT 'pending',  -- 'pending' | 'certified' | 'expired'
    created_at DATETIME,
    updated_at DATETIME,
    is_new BOOLEAN DEFAULT FALSE
);
```

### realname_auth 表（实名认证记录）
```sql
CREATE TABLE realname_auth (
    id INT PRIMARY KEY AUTO_INCREMENT,
    uid INT NOT NULL REFERENCES users(uid),
    name VARCHAR(50) NOT NULL,
    idcard VARCHAR(18) NOT NULL,
    phone VARCHAR(11) NOT NULL,
    code VARCHAR(6),
    verified BOOLEAN DEFAULT FALSE,
    verified_at DATETIME,
    created_at DATETIME
);
```

### employee_auth 表（在职认证记录）
```sql
CREATE TABLE employee_auth (
    id INT PRIMARY KEY AUTO_INCREMENT,
    uid INT NOT NULL REFERENCES users(uid),
    company VARCHAR(100) NOT NULL,
    position VARCHAR(50) NOT NULL,
    industry VARCHAR(50) NOT NULL,
    verified BOOLEAN DEFAULT FALSE,
    verified_at DATETIME,
    created_at DATETIME
);
```

### material_auth 表（材料认证记录）
```sql
CREATE TABLE material_auth (
    id INT PRIMARY KEY AUTO_INCREMENT,
    uid INT NOT NULL REFERENCES users(uid),
    method VARCHAR(20) NOT NULL,  -- 'email' | 'contract' | 'tax' | 'other'
    email VARCHAR(100),
    file_path VARCHAR(200),
    verified BOOLEAN DEFAULT FALSE,
    verified_at DATETIME,
    created_at DATETIME
);
```

### audit_log 表（审核日志）
```sql
CREATE TABLE audit_log (
    id INT PRIMARY KEY AUTO_INCREMENT,
    uid INT NOT NULL REFERENCES users(uid),
    reviewer_id INT REFERENCES users(uid),
    action VARCHAR(20),  -- 'approve' | 'reject'
    remark VARCHAR(500),
    created_at DATETIME
);
```

### user_cert_status 表（用户认证状态 - 新增）
```sql
CREATE TABLE user_cert_status (
    uid INT PRIMARY KEY REFERENCES users(uid),
    status VARCHAR(30) NOT NULL DEFAULT 'not-certified',  
    -- 'not-certified' | 'basic-certified' | 'specialist-certified'
    cert_count INT DEFAULT 0,  -- 认证次数
    current_cert_type VARCHAR(50),  -- 当前认证类型（企业职工 或 具体专业职称）
    last_cert_time DATETIME,  -- 最后一次认证时间
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### certification_history 表（认证历史记录 - 新增）
```sql
CREATE TABLE certification_history (
    id INT PRIMARY KEY AUTO_INCREMENT,
    uid INT NOT NULL REFERENCES users(uid),
    cert_number INT NOT NULL,  -- 第几次认证（1, 2, 3...）
    cert_type VARCHAR(50) NOT NULL,  
    -- 'basic' | 'specialist_lawyer' | 'specialist_design' | 'specialist_media' | 'specialist_dev'
    is_upgrade BOOLEAN DEFAULT FALSE,  -- 是否为升级认证
    
    -- 认证信息
    company VARCHAR(100),  -- 企业名称
    position VARCHAR(50),  -- 职位
    industry VARCHAR(50),  -- 行业
    auth_method VARCHAR(20),  -- 认证方式：'email' | 'contract' | 'tax' | 'other'
    
    -- 材料信息
    email VARCHAR(100),  -- 企业邮箱（如果是邮箱认证）
    file_path VARCHAR(200),  -- 上传文件路径
    file_name VARCHAR(100),  -- 上传文件名
    
    -- 审核信息
    review_status VARCHAR(20) DEFAULT 'pending',  
    -- 'pending' | 'approved' | 'rejected'
    reviewer_id INT REFERENCES users(uid),  -- 审核人ID
    review_remark VARCHAR(500),  -- 审核意见
    
    -- 时间戳
    submitted_at DATETIME NOT NULL,  -- 用户提交时间
    reviewed_at DATETIME,  -- 审核时间
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    FOREIGN KEY (uid) REFERENCES users(uid),
    INDEX idx_uid (uid),
    INDEX idx_submitted_at (submitted_at),
    INDEX idx_review_status (review_status)
);
```
